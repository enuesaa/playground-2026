# jj

- git の代替
- git と同じインタフェースらしい
  - サブコマンドとかが違う感じ？
  - 既存のgitレポではまず`jj git init --colocate`しないと使えなさそう
- パッと触った感じ、全然わからん
  - 真面目にドキュメント読まないとダメそう。
  - あと git ってソフトウェア開発には必須の存在だから、その世界から外れたところに行く感覚があって怖い
- 記事を見ている感じ、コンフリクト解決の考え方とかは jj の方が for me に思える
- .git とは別に .jj というディレクトリができる
- jj 触った後に git を触ると、なんか頭が混乱するのでどっちかにした方がいいかも
