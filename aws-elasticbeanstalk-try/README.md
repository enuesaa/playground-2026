# AWS Elastic Beanstalk

- AWSの試験でいつも出てくるやつ
- まともに触ったのは初めて
- AWS EC2 & Auto Scaling の仕組みに乗っかって、インフラの構築やアプリケーションをデプロイしてくれる
  - Elastic Beanstalk 自体では料金かからない
  - EC2 や S3 とかでお金がかかる
- リソースを作成するときに VPC を選んだりセキュリティグループを選んだり Public IP アドレスを有効化するか聞かれる
  - EC2 の知識ないとこのあたり判断できなさそう。
- リソースを作成すると S3 バケットが作成される
  - アプリケーションコードは一度ここにアップロードされる？っぽい
  - あと EC2 からアップロード？もするみたい
  - EC2 のインスタンスプロファイルに S3 の権限が必要
- AWSの試験でいつも出てくるけど、デプロイするときにオプションというか戦略みたいなのあるからそれを何にするか検討必要
  - https://dev.classmethod.jp/articles/elastic-beanstalk-deploy-policy/
- 「環境を再構築」というボタンで EC2 インスタンスを立ち上げ直しできる
- Elsatic Beanstalk のログはコンソールから確認できる
  - EC2 の /var/log にもある
- 言語は PHP を選んだ
  - 初めはサンプルアプリをデプロイできるよう。
    - index.php があるだけのシンプルなアプリだった
  - ウェブサーバーは Apache を選択
- 環境用のドメインが発行される
- そういえば docker コンテナのデプロイもできるっぽい。
